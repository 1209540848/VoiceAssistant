# 桌面端 UI 改版记录

## 本次改了什么

### 1. 主窗口从工程台改成桌面输入法控制台

旧版把录音、热词、实时转写、诊断都放在一个页面里，像调试台。

新版改成左侧导航：

- 首页
- 历史记录
- 词典
- 设置
- 反馈

首页只保留高频输入相关内容：快捷键、录音入口、使用统计、最后一次输入。

### 2. 词典独立成页面

热词不再挤在录音旁边。

用户可以在“词典”页维护自己的热词：

- 一行一个热词
- 保存到本地
- 同步到 DashScope 热词列表
- 显示热词数量和预览

用户热词保存到 `data/user-hotwords.json`，该文件已加入 `.gitignore`。

### 3. 新增用户热词 API

新增接口：

- `GET /api/hot-words`
- `POST /api/hot-words`
- `POST /api/hot-words/sync`

同步成功后会保存 DashScope 返回的 `vocabulary_id`，后续 Fun-ASR 请求自动带上该 ID。

### 4. 新增历史记录和统计

前端新增本地历史记录：

- 最近 20 条转写
- 今日口述字数
- 今日口述时间
- 平均口述速度
- 用户热词数量

当前先用 `localStorage`，后续可以迁移到本地文件或数据库。

### 5. 诊断信息移到设置页

Trace、Request、延迟、错误不再占用首页空间。

它们被移到“设置”页，保留排查能力，但不干扰普通使用路径。

### 6. 视觉风格调整

参考 Typeless 的产品结构，改成更轻的桌面软件风格：

- 浅灰背景
- 左侧导航
- 白色内容区
- 低对比边框
- 减少阴影和技术标签
- 强调快捷键和最后一次输入

## 交互问题记录：切换页面后底部露灰

### 问题

点击“设置”“反馈”等内容较少的页面时，右侧白色内容区高度会缩短，底部露出灰色背景，视觉上像页面断层。

### 原因

上一次为了减少首页底部空白，把 `.desktop-shell`、`.content-shell`、`.page-view` 的高度从视口撑满改成了内容自适应。

这个改法在首页内容较多时看起来没问题，但切换到短页面时，右侧内容区只包住实际内容，无法和左侧侧栏保持一致高度，于是底部露出应用背景色。

### 第一次修复为什么不够

第一次尝试使用固定最小高度：

- `.desktop-shell` / `.content-shell` 固定为 `560px`
- `.page-view` 固定为 `500px`

这个方案只避免了右侧白色内容区塌缩，但 Electron 主窗口本身仍然比短页面内容高，所以底部还是会露出一块灰色空白，没有满足“窗口贴合内容”的诉求。

### 第二次修复为什么也不对

第二次尝试让 Electron 主窗口根据当前页面高度动态 `setContentSize()`。

这个方向是错的：桌面应用主窗口不应该因为点击“首页 / 词典 / 设置 / 反馈”而变高变矮。动态窗口高度会导致每次切页时窗口抖动，用户感知更差。

### 最终解决

统一应用画布尺寸：

1. Electron 主窗口固定为 `1180 x 640`，最小高度也是 `640`。
2. `.desktop-shell`、`.sidebar`、`.content-shell` 使用统一 `100vh` 高度。
3. `.page-view` 使用统一 `height: 100%`。
4. 页面内容超出时在右侧内容区内部滚动，不改变窗口大小。
5. 删除 `resizeMainWindow`、`desktop-window:resize-main`、`scheduleMainWindowResize` 等动态缩放逻辑。

这样切换不同页面时窗口不会抖动，左侧侧栏和右侧内容区始终同高。短页面保留统一白色画布，长页面内部滚动。

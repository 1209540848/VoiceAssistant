# 桌面端 UI 改版记录

## 本次改了什么

### 1. 主窗口从工程台改成桌面输入法控制台

旧版把录音、热词、实时转写、诊断都放在一个页面里，像调试台。

新版改成左侧导航：

- 首页
- 历史记录
- 词典
- 设置
- 反馈

首页只保留高频输入相关内容：快捷键、录音入口、使用统计、最后一次输入。

### 2. 词典独立成页面

热词不再挤在录音旁边。

用户可以在“词典”页维护自己的热词：

- 一行一个热词
- 保存到本地
- 同步到 DashScope 热词列表
- 显示热词数量和预览

用户热词保存到 `data/user-hotwords.json`，该文件已加入 `.gitignore`。

### 3. 新增用户热词 API

新增接口：

- `GET /api/hot-words`
- `POST /api/hot-words`
- `POST /api/hot-words/sync`

同步成功后会保存 DashScope 返回的 `vocabulary_id`，后续 Fun-ASR 请求自动带上该 ID。

### 4. 新增历史记录和统计

前端新增本地历史记录：

- 最近 20 条转写
- 今日口述字数
- 今日口述时间
- 平均口述速度
- 用户热词数量

当前先用 `localStorage`，后续可以迁移到本地文件或数据库。

### 5. 诊断信息移到设置页

Trace、Request、延迟、错误不再占用首页空间。

它们被移到“设置”页，保留排查能力，但不干扰普通使用路径。

### 6. 视觉风格调整

参考 Typeless 的产品结构，改成更轻的桌面软件风格：

- 浅灰背景
- 左侧导航
- 白色内容区
- 低对比边框
- 减少阴影和技术标签
- 强调快捷键和最后一次输入

## 当前判断

这一版把产品从“语音识别 Demo”推进到“桌面语音输入助手控制台”。

下一步应该继续补：

- 自动粘贴到当前输入框
- 快捷键自定义
- 历史记录持久化到本地文件
- 用户热词同步失败时的更清晰提示
